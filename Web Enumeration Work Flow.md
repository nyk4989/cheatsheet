# # SQLi
## ## 1. URLパラ、POSTパラ、JSONボディ
- 怪しい値(id / user / page)などのDBとやり取りしてそうな値がある場合にSQLiを試す。

```例
https://0afc009d047790c78112d44600d300c9.web-security-academy.net/filter?category=Gifts
```
- [ ] `'`シングルクォーテーションを入力。
	- [ ] レスポンスに差異がない。
	- [ ] SQLの構文エラーがない。
- [ ] `"`ダブルクォーテーションを入力。
	- [ ] レスポンスに差異がない。
	- [ ] SQLの構文エラーがない。
